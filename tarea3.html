Comenzar
HTML5 (HyperText Markup Language versión 5) es el lenguaje fundamental de la web. Con él defines la estructura y el significado de cada elemento de una página: títulos, párrafos, imágenes, videos, formularios y más. Esta guía te lleva desde cero absoluto hasta dominar todas las etiquetas y conceptos esenciales, paso a paso, con ejemplos reales.
0

¿Qué es HTML y cómo funciona?

Conceptos base antes de escribir la primera línea

HTML no es un lenguaje de programación: es un lenguaje de marcado. Funciona usando etiquetas (tags) que le indican al navegador cómo mostrar el contenido. Cada etiqueta envuelve un fragmento de contenido y le da significado.

El navegador (Chrome, Firefox, Safari) lee tu archivo .html, interpreta las etiquetas y renderiza la página visual que ves en pantalla.

Solo necesitas un editor de texto (como VS Code, que es gratuito) y un navegador web. ¡No se necesita instalar nada más para comenzar!
1

Estructura básica de un documento HTML5

El esqueleto que todo archivo HTML debe tener

Todo documento HTML5 tiene una estructura base obligatoria. Es como los cimientos de una casa: sin ellos, nada funciona correctamente. Esta estructura le dice al navegador qué tipo de archivo es, en qué idioma está y cómo está organizado.

mi-primera-pagina.html
<!-- Paso 1: estructura mínima obligatoria --> <!DOCTYPE html> <html lang="es"> <head> <!-- Metadatos: información SOBRE la página --> <meta charset="UTF-8"/> <meta name="viewport" content="width=device-width, initial-scale=1.0"/> <title>Mi Primera Página Web</title> </head> <body> <!-- Todo lo que el usuario VE va aquí --> <h1>¡Hola Mundo!</h1> <p>Mi primera página con HTML5.</p> </body> </html>
<!DOCTYPE html>
Declara que el documento usa HTML5. Siempre va en la primera línea.
<html>
Elemento raíz. Todo el documento vive dentro de él. El atributo lang indica el idioma.
<head>
Contiene metadatos: título, charset, enlaces a CSS, SEO. No es visible en pantalla.
<body>
Todo lo que el usuario ve en pantalla va aquí: textos, imágenes, botones, etc.
<meta charset>
Define la codificación de caracteres. UTF-8 permite tildes y ñ correctamente.
<title>
Texto que aparece en la pestaña del navegador y en resultados de búsqueda.
2

Etiquetas de Texto y Tipografía

Títulos, párrafos, énfasis y contenido semántico

Las etiquetas de texto son las más usadas. Con ellas defines jerarquía de contenido, párrafos, palabras importantes y más. HTML5 enfatiza el uso semántico: usar la etiqueta correcta para cada tipo de contenido.

textos.html
<!-- Títulos: h1 es el más importante --> <h1>Título principal (solo 1 por página)</h1> <h2>Sección importante</h2> <h3>Sub-sección</h3> <!-- Párrafo --> <p>Este es un párrafo. Puede contener texto largo con negritas y cursivas mezcladas.</p> <!-- Énfasis semántico --> <p>Palabra <strong>muy importante</strong> en negrita.</p> <p>Término <em>enfatizado</em> en cursiva.</p> <!-- Otras etiquetas de texto --> <p>Texto <mark>resaltado</mark> en amarillo.</p> <p>Precio: <del>$200</del> <ins>$150</ins></p> <p>H<sub>2</sub>O y E=mc<sup>2</sup></p> <blockquote>Una cita larga destacada del texto.</blockquote> <code>console.log("código inline")</code> <br/> <!-- salto de línea --> <hr/> <!-- línea horizontal divisora -->
<h1> – <h6>
Jerarquía de títulos. h1 es el más importante. Solo debe haber un h1 por página.
<p>
Párrafo de texto. Es un elemento de bloque: genera salto antes y después.
<strong>
Texto muy importante (negrita). Tiene valor semántico SEO.
<em>
Énfasis (cursiva). Indica estrés o acento en la lectura.
<mark>
Resalta texto como con un marcador amarillo.
<blockquote>
Cita larga de otra fuente. Se muestra sangrada.
<del> / <ins>
Texto eliminado (tachado) e insertado. Útil para precios y cambios.
<sub> / <sup>
Subíndice (H₂O) y superíndice (E=mc²).
<br/>
Salto de línea forzado. Etiqueta vacía (sin cierre).
<hr/>
Línea horizontal divisora. Etiqueta vacía.
<code>
Código inline con fuente monoespaciada.
<pre>
Texto preformateado: respeta espacios y saltos de línea.

Listas: ordenadas, desordenadas y de definición

Organiza la información de forma estructurada

Las listas son fundamentales para organizar información. HTML ofrece tres tipos: listas sin orden (<ul>), listas numeradas (<ol>) y listas de definición (<dl>) para términos y descripciones.

listas.html
<!-- Lista Desordenada (bullets) --> <ul> <li>HTML – estructura</li> <li>CSS – estilos <ul> <li>SASS – preprocesador</li> </ul> </li> <li>JavaScript – interactividad</li> </ul> <!-- Lista Ordenada (numerada) --> <ol start="1"> <li>Aprender HTML</li> <li>Aprender CSS</li> <li>Aprender JavaScript</li> </ol> <!-- Lista de Definición (término + descripción) --> <dl> <dt>HTML</dt> <dd>Lenguaje de marcado para la web</dd> <dt>CSS</dt> <dd>Lenguaje de estilos en cascada</dd> </dl>
4

Enlaces e Imágenes

Conecta páginas y muestra contenido visual

Los hipervínculos son la esencia de la web: conectan páginas entre sí. Las imágenes son el contenido visual más básico. Ambos usan atributos clave que debes memorizar.

enlaces-imagenes.html
<!-- ── ENLACES ── --> <!-- Enlace externo, abre en nueva pestaña --> <a href="https://google.com" target="_blank" rel="noopener"> Ir a Google </a> <!-- Enlace a otra página local --> <a href="contacto.html">Contacto</a> <!-- Ancla interna (navega a un id) --> <a href="#seccion1">Ir a Sección 1</a> <a href="mailto:hola@correo.com">Enviar email</a> <a href="tel:+521234567890">Llamar</a> <!-- Imagen como enlace --> <a href="https://ejemplo.com"> <img src="logo.png" alt="Logo"/> </a> <!-- ── IMÁGENES ── --> <img src="fotos/mi-foto.jpg" alt="Descripción accesible de la imagen" width="600" height="400" loading="lazy" /> <!-- Imagen responsive con múltiples fuentes --> <picture> <source srcset="imagen.webp" type="image/webp"/> <img src="imagen.jpg" alt="Fallback"/> </picture> <!-- Figura con pie de imagen --> <figure> <img src="grafico.png" alt="Gráfico de datos"/> <figcaption>Fig. 1 – Resultados del estudio</figcaption> </figure>
El atributo alt en imágenes es obligatorio para accesibilidad. Los lectores de pantalla lo leen en voz alta para personas con discapacidad visual. También aparece si la imagen no carga.
5

Etiquetas Semánticas HTML5

Estructura con significado: la gran novedad de HTML5

HTML5 introdujo etiquetas semánticas: en lugar de usar <div> para todo, ahora usamos etiquetas con significado propio. Esto beneficia el SEO, la accesibilidad y la legibilidad del código.

semantica.html
<body> <header> <img src="logo.png" alt="Mi Marca"/> <nav> <a href="/">Inicio</a> <a href="/blog">Blog</a> <a href="/contacto">Contacto</a> </nav> </header> <main> <article> <h1>Título del artículo</h1> <section> <h2>Primera sección</h2> <p>Contenido...</p> </section> </article> <aside> <h3>Artículos relacionados</h3> </aside> </main> <footer> <p>&copy; 2025 Sophia Castro</p> <address>contacto@sophia.com</address> </footer> </body>
<header>
Encabezado de página o sección. Suele contener el logo y navegación.
<nav>
Bloque de navegación principal. Solo para grupos de enlaces de navegación.
<main>
Contenido principal único de la página. Solo debe haber uno por documento.
<article>
Contenido independiente y reutilizable: posts, noticias, comentarios.
<section>
Agrupa contenido temáticamente relacionado. Suele llevar un título h2.
<aside>
Contenido lateral complementario al principal: sidebar, widgets, notas.
<footer>
Pie de página con derechos, contacto, mapa del sitio.
<div>
Contenedor genérico sin semántica. Usarlo cuando ninguna etiqueta semántica aplique.
<span>
Contenedor inline genérico. Para dar estilo a partes del texto sin semántica.
<address>
Información de contacto del autor o de la empresa.
<time>
Fecha u hora legible por máquinas con el atributo datetime.
<details> / <summary>
Acordeón nativo HTML: contenido desplegable sin JavaScript.

Tablas de Datos

Presenta información organizada en filas y columnas

Las tablas HTML son para mostrar datos tabulares (como hojas de cálculo). No deben usarse para maquetar layouts. Tienen una estructura jerárquica: cabecera, cuerpo y pie.

tabla.html
<table> <!-- Cabecera --> <thead> <tr> <th scope="col">Nombre</th> <th scope="col">Materia</th> <th scope="col">Calificación</th> </tr> </thead> <!-- Cuerpo --> <tbody> <tr> <td>Sophia Castro</td> <td>Cultura Digital</td> <td>10</td> </tr> <!-- Celda que ocupa 2 columnas --> <tr> <td colspan="2">Promedio general</td> <td>9.8</td> </tr> </tbody> <!-- Pie --> <tfoot> <tr> <td colspan="3">Total estudiantes: 3</td> </tr> </tfoot> </table>

Formularios HTML5

Recolecta datos del usuario: el corazón de la interactividad

Los formularios permiten que los usuarios envíen datos: registros, búsquedas, pagos, comentarios. HTML5 amplió enormemente los tipos de input, añadiendo validación nativa sin JavaScript.

formulario.html
<form action="/enviar" method="post"> <!-- Agrupación con fieldset --> <fieldset> <legend>Datos personales</legend> <label for="nombre">Nombre *</label> <input type="text" id="nombre" name="nombre" placeholder="Tu nombre" required minlength="2"/> <label for="email">Email *</label> <input type="email" id="email" name="email" placeholder="hola@ejemplo.com" required/> <label for="edad">Edad</label> <input type="number" id="edad" min="14" max="120"/> <label for="nacimiento">Fecha de nacimiento</label> <input type="date" id="nacimiento"/> </fieldset> <!-- Select desplegable --> <label for="especialidad">Especialidad</label> <select id="especialidad" name="especialidad"> <option value="">-- Selecciona --</option> <option value="ia">Inteligencia Artificial</option> <option value="prog">Programación</option> </select> <!-- Textarea --> <label for="mensaje">Mensaje</label> <textarea id="mensaje" rows="4" cols="40" placeholder="Escribe aquí..."></textarea> <!-- Botones --> <button type="submit">Enviar</button> <button type="reset">Limpiar</button> </form>
8

Multimedia: Audio, Video e iFrame

Incrusta contenido audiovisual sin plugins

HTML5 revolucionó la web al permitir audio y video nativos sin necesitar Flash ni plugins. Con <video> y <audio> puedes controlar la reproducción directamente desde HTML. <iframe> permite incrustar páginas externas.

multimedia.html
<!-- VIDEO nativo HTML5 --> <video width="640" height="360" controls autoplay muted loop poster="portada-video.jpg"> <source src="video.mp4" type="video/mp4"/> <source src="video.webm" type="video/webm"/> Tu navegador no soporta video. </video> <!-- AUDIO nativo HTML5 --> <audio controls> <source src="cancion.mp3" type="audio/mpeg"/> <source src="cancion.ogg" type="audio/ogg"/> </audio> <!-- IFRAME – Incrustar YouTube --> <iframe src="https://www.youtube.com/embed/ID_VIDEO" width="560" height="315" allowfullscreen loading="lazy" title="Video de YouTube"> </iframe> <!-- CANVAS – Dibujo con JavaScript --> <canvas id="miCanvas" width="500" height="300"></canvas>

Atributos Globales y Buenas Prácticas

Los atributos que funcionan en cualquier etiqueta

Los atributos globales se pueden aplicar a casi cualquier etiqueta HTML. Son esenciales para el CSS, JavaScript, accesibilidad y SEO. Dominarlos es lo que diferencia un código básico de uno profesional.

Atributo Para qué sirve Ejemplo de uso
idIdentifica un elemento único en la página. Usado por CSS (#id) y JS<div id="menu">
classAgrupa elementos para estilos CSS o selectores JS. Puede repetirse<p class="destacado">
styleCSS inline directamente en el elemento (no recomendado para producción)<h1 style="color:red">
titleTooltip al pasar el mouse encima del elemento<abbr title="HyperText">HTML</abbr>
langIdioma del contenido del elemento<p lang="en">Hello</p>
hiddenOculta el elemento visualmente y del DOM accesible<div hidden>
tabindexOrden de navegación con teclado (accesibilidad)<button tabindex="1">
data-*Atributos personalizados para guardar datos en HTML<div data-usuario="sophia">
contenteditableHace el elemento editable por el usuario<p contenteditable="true">
draggablePermite arrastrar el elemento (Drag & Drop)<div draggable="true">
aria-labelEtiqueta accesible para lectores de pantalla<button aria-label="Cerrar">
roleDefine el rol ARIA del elemento para accesibilidad<div role="dialog">

Buenas prácticas HTML5 — checklist del desarrollador:

  • Siempre incluir <!DOCTYPE html> al inicio del archivo
  • Usar lang="es" en <html> para SEO y accesibilidad
  • Incluir siempre el atributo alt en todas las imágenes
  • Usar etiquetas semánticas (<header>, <main>, <footer>) en lugar de <div> para todo
  • Solo usar un <h1> por página; respetar la jerarquía h1 → h2 → h3
  • Indentar correctamente el código (2 o 4 espacios por nivel)
  • Cerrar todas las etiquetas (excepto las vacías: <img/>, <br/>, <hr/>)
  • Incluir <meta name="viewport"> para que la web sea responsive en móviles
  • No usar estilos inline (style=""). Mantener HTML y CSS separados
  • Validar el código en validator.w3.org antes de publicar

Resumen: Todo HTML5 en un vistazo

Lo que aprendiste en esta guía

🏗️ Estructura Base

DOCTYPE, html, head, body. El esqueleto obligatorio de todo documento.

📝 Texto y Tipografía

h1-h6, p, strong, em, mark, blockquote, code y más etiquetas de contenido.

📋 Listas

ul, ol, dl — tres tipos para organizar cualquier información estructurada.

table, thead, tbody, tfoot, tr, th, td. Para datos tabulares nunca para layout.

🎬 Multimedia

video, audio, iframe, canvas — contenido audiovisual sin Flash ni plugins.

🌐 Atributos Globales

id, class, data-*, aria-*, style, hidden — aplican en cualquier etiqueta.
